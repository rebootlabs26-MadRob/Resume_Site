Network Security Implementation

Dec 2025 - Present

Built layered security setup with DNS filtering (Pi-hole), intrusion detection (Suricata), SSH protection (fail2ban), and firewall rules (UFW). All security events feed into Loki for centralized logging. Currently setting up WireGuard VPN for secure remote access.

Security Results: Blocks over 40% of malicious DNS requests, zero successful intrusions, real-time alerts for suspicious activity

Implementation Details:

  • Pi-hole: Network-wide DNS filtering blocking over 40% of malicious requests
  • Suricata IDS: Signature-based detection with Emerging Threats ruleset
  • fail2ban: Adaptive SSH brute-force protection with escalating ban times
  • UFW Firewall: Default-deny policy with granular service-level rules
  • WireGuard VPN: Split-tunnel remote access with automatic DNS routing

Hardware Diagnostic Platform

2023 - Present

Built a custom diagnostic tool using Raspberry Pi that reads POST codes during computer boot to catch hardware faults. Once the system is running, it switches to monitoring CPU temp, memory, disk health, and network activity. Added security scanning to detect malware and corrupted data. Designed for fast motherboard diagnostics and ongoing system health checks during repairs.

Technical Features:

  • GPIO POST code capture for real-time boot diagnostics
  • Dual-mode operation: boot fault detection → system health monitoring
  • Custom Tkinter interface with cyberpunk theming and adaptive scaling
  • Integrated security scanning (malware detection, file integrity checks)
  • Real-time system metrics (CPU temp, memory, disk I/O, network stats)
  • Optimized for 320x480 TFT displays and Raspberry Pi hardware

Docker Infrastructure Migration

Dec 2025

Executed complete containerization platform migration from Snap to Docker CE. Implemented backup/restore procedures for 7+ service containers, validated data integrity at each migration step, and documented the process for reproducibility. Eliminated permission conflicts and improved container restart reliability.

Migration Process:

  • Pre-migration: Documented existing Snap configurations and data paths
  • Backup: Created complete backups of all service data and configurations
  • Migration: Deployed Docker Compose stacks with equivalent configurations
  • Validation: Verified data integrity and service functionality post-migration
  • Optimization: Improved restart policies and resource allocation
  • Documentation: Created reproducible migration guide for future reference
